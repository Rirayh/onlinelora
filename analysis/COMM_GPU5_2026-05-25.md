
### 原因
vllm 的启动序列是：
1. CPU init (engine config, parallel state) — 0 GPU mem
2. NCCL init — 几 MB
3. Load safetensors (CPU → GPU, mistral-7b ~14 GB) — 这一步会从 0 → 14GB **持续 60-120 秒**
4. KV cache alloc (gpu_memory_utilization=0.85) — 再爬升到 ~70 GB
5. CUDA graph capture (51 size buckets) — 30-60 秒
6. 开始评测

**你看到 33MB 的瞬间 = 步骤 2-3 之间的过渡**。这种情况在 P0 reeval 里大概每个 cell 都会出现一次（持续 ~1 分钟）。
